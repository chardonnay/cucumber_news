1 Introduction

What is Cucumber NewsScraper?

Cucumber NewsScraper is a responsive, client-side news dashboard that aggregates articles from many publishers into one clean interface. It can generate optional AI summaries for each article, find related discussions and videos, translate content, and sort by recency, dates, comment activity, or reader ratings.

It runs in your browser. A small bundled Python server (called the dev server) serves the page and proxies news feeds and AI requests so the browser never hits cross-origin (CORS) problems. Everything you configure — sources, summaries, settings — is stored locally in your browser (IndexedDB and localStorage). There are no accounts and no telemetry.

Key features at a glance

  • Many sources — dedicated integrations (heise.de, Telepolis, Golem, ComputerBase, t3n, IT-Administrator, BILD, The Verge) plus a large international registry (Reuters, AP, BBC, CNN, NYT, Guardian, Bloomberg, Der Spiegel, Le Monde, Al Jazeera and many more).
  • Optional AI summaries via LM Studio (local), any OpenAI-compatible server, or the Anthropic Messages API.
  • Dynamic category filters, and sorting by recency, date range, comment counts, or green/red ratings.
  • Alternative article links, related Reddit threads, and YouTube suggestions per article.
  • Machine translation of the article area, PDF/CSV/JSON export, and an AI usage statistics panel.
  • Deep theming — light/dark, eight accent colors, brightness and transparency sliders, and custom colors.
  • Bilingual UI (German / English).

How to use this manual

  • Search — type in the search box (top). Results show the chapter and section that match, with a snippet; click to jump straight there.
  • Ask the AI — click Ask the AI to ask a question in plain language. The assistant answers from this manual first (and tells you when it falls back to general knowledge). It reuses whatever AI backend you already configured in the app.
  • Browse — use the Contents list on the left.

2 Quick Start

Run it in three commands

If Python 3 is already installed, this is the whole thing:

# 1) go to the project folder
cd news_reader_web

# 2) install the one dependency
pip install requests

# 3) start the bundled server
python3 scripts/dev_server.py

Then open http://127.0.0.1:8080/index.html in your browser. That's it — you'll see news right away.

Want AI summaries too? After the app loads, open KI-Server in the header and pick a mode. See AI / KI Setup.

WindowsOn Windows the command is usually python scripts\dev_server.py (with a backslash). Full step-by-step in Installation → Windows.

3 Installation

Prerequisites

  • Python 3.8 or newer — the only thing you must install. Check with python3 --version (macOS/Linux) or python --version (Windows).
  • pip — Python's package installer, bundled with modern Python. Used once to install requests.
  • A modern browser — Chrome, Edge, Firefox, or Safari.
  • Optional: LM Studio for local AI, or an Anthropic API key for cloud AI.

No Node.js, no build step, no npm — the frontend is plain HTML + vanilla JavaScript served as-is.

WindowsInstall on Windows (step by step)

  1. Install Python. Download the latest installer from python.org/downloads/windows. In the installer, check “Add python.exe to PATH” on the first screen, then click Install Now.

    If you forget “Add to PATH”, re-run the installer, choose Modify, and enable it — or use the py launcher (e.g. py scripts\dev_server.py), which is installed regardless.

  2. Open a terminal. Press Win + R, type powershell, press Enter. (Or search “PowerShell” / “Terminal” in the Start menu.)
  3. Verify Python: 
    python --version
# if that fails, try:
py --version
  4. Go to the project folder (adjust the path to where you unzipped it): 
    cd C:\Users\YourName\Downloads\news_reader_web
  5. (Recommended) Create a virtual environment so requests stays isolated: 
    python -m venv venv
.\venv\Scripts\Activate.ps1

    If PowerShell says “running scripts is disabled on this system”, allow local scripts for your user once:

    Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

    Then run the Activate.ps1 line again. (In classic Command Prompt the activate command is venv\Scripts\activate.bat instead.)

  6. Install the dependency: 
    pip install requests
  7. Start the server: 
    python scripts\dev_server.py
  8. Open the app: go to http://127.0.0.1:8080/index.html.

Port already in use? Find the process and either stop it or pick another port:

netstat -ano | findstr ":8080"
python scripts\dev_server.py --port 9000

Then open http://127.0.0.1:9000/index.html.

Stuck on a Windows step? See the curated External Resources — they link straight to the official Python, PATH, venv, and PowerShell help pages.

The one dependency: requests

The dev server relies on stdlib for almost everything; the only external package is requests (used for robust feed fetching). Install it with pip install requests (or pip3).

Seeing ModuleNotFoundError: No module named 'requests'? You installed it for a different Python than the one running the server. If you use a virtual environment, make sure it's activated before both the pip install and the python command.

Starting the dev server (options)

The default just works:

python3 scripts/dev_server.py

You can override the defaults with command-line flags:

FlagDefaultPurpose
--port8080Port the app is served on.
--listen127.0.0.1Bind address. Use 0.0.0.0 to reach it from another device on your LAN.
--targethttp://127.0.0.1:1234Where LM Studio is running (the server proxies /api/v1/chat to it).
 
python3 scripts/dev_server.py --port 9000 --listen 0.0.0.0 --target http://192.168.1.50:1234

Plain static server (limited — usually not what you want)

You can serve the files with Python's built-in server:

python3 -m http.server 8080

…but this skips all the /api/* routes. Without the dev server you lose feed proxying (many feeds fail in the browser due to CORS), same-origin AI proxying, comment counts, alternative links, and the whole international source catalog (which depends on /api/source-feed). Prefer scripts/dev_server.py.

Verify it works

  • The terminal prints something like Open: http://127.0.0.1:8080/index.html when the server starts.
  • Open that URL — articles from Heise (and other enabled sources) should appear within a few seconds.
  • If the grid is empty or shows only a demo item, open the browser console (F12) and check Troubleshooting.

4 AI / KI Setup

The three AI modes

Open KI-Server in the app header and choose an API mode. All AI features (summaries, Reddit query optimization, YouTube suggestions — and this manual's Ask the AI) use whichever mode you pick.

ModeBest forEndpointNeeds a key?
LM Studio REST v1 (default)Free, private, local modelsPOST /api/v1/chatNo (local)
OpenAI-compatibleOllama, vLLM, LocalAI, OpenAI, etc.POST /v1/chat/completionsDepends on server
Anthropic Messages APICloud Claude modelsPOST /v1/messagesYes

LM Studio (recommended local setup)

  1. Install LM Studio, download a model, and start its local server (usually port 1234).
  2. In the app open KI-Server → API mode LM Studio REST v1.
  3. Set Server URL to http://127.0.0.1:1234 (server root only — no path).
  4. Leave Model on Automatic to use whatever model is currently loaded in LM Studio, or pick a specific one from the list.
  5. Enable “REST via same page origin”, then Save and reload.

Why “REST via same page origin”? It routes AI calls through the dev server (/api/v1/chat) instead of the browser calling LM Studio directly. This avoids the OPTIONS preflight that many LM Studio builds reject — the usual cause of “Failed to fetch”. It requires scripts/dev_server.py to be the page you opened.

Anthropic Messages API (cloud Claude)

  1. Create an API key in the Anthropic Console.
  2. Open KI-Server → API mode Anthropic Messages API.
  3. Set Server URL to https://api.anthropic.com/v1 (the client appends /messages and /models).
  4. Paste your API key (required — sent as the x-api-key header).
  5. Leave Model on Automatic (uses the first model the API returns) or select one.

Your key is stored only in your browser's localStorage and sent directly to Anthropic. Reasoning controls are hidden in this mode because the app sends only the standard Messages fields (model, system, messages, temperature, max_tokens, stream).

OpenAI-compatible servers

For Ollama, vLLM, LocalAI, or the real OpenAI API, choose mode OpenAI-compatible. The Server URL must end in /v1 (e.g. http://127.0.0.1:1234/v1) — not …/v1/chat/completions; the client appends the path itself. Provide an API token only if your server requires one.

A remote OpenAI-compatible host (anything other than 127.0.0.1/localhost) can't be reached from this manual page's Ask the AI due to its Content-Security-Policy. Article summaries in the app are unaffected — only the manual's AVE box is constrained. See Troubleshooting → CSP.

Model selection & reasoning

Automatic vs. manual: with Automatic, the app resolves the currently loaded model before each request (great when you swap models in LM Studio). Pick a model to pin it. Use the ↻ button to reload the list.

Reasoning (LM Studio mode only): the app reads each model's capabilities.reasoning.allowed_options from the native model list and only shows the levels that model actually supports. Models without reasoning support show only Off. If the app can reach only the older OpenAI-style model list (no capability metadata), it conservatively offers Off. Reasoning controls are hidden entirely in OpenAI-compatible and Anthropic modes.

Summary language, style & performance

  • Language — Site matches the article's source language (German for .de, English for The Verge); Browser uses your browser language.
  • Style — Factual, Professional, Casual, or Custom (your own free-text instruction, up to 300 characters).
  • Summary cache (days) — how long summaries are reused from IndexedDB. 0 = never expire.
  • Concurrency — how many summaries are generated in parallel during batch/auto runs (1–16, default 4). Higher = faster but heavier on your model.
  • Request timeout (seconds) — abort an AI call after this long (default 120).

See the full list of knobs in the Configuration Reference.

5 Using the App

Sources: select, favorite, enable

  • Switch source with the Quelle (Source) dropdown in the header. Enabled sources are listed alphabetically; use the ◄ ► buttons to step through them.
  • Favorite the active source with the star next to its logo. A separate favorites-jump button cycles only your favorites.
  • Choose which sources appear in Einstellungen (Settings) → sources list. Filter by name, bulk enable/disable, and keep at least one enabled. The huge international catalog (Reuters, BBC, etc.) lives here too.

Heise magazines / sub-feeds

Under Einstellungen → Extra heise.de-Bereiche you can enable additional Heise magazine feeds (iX, c't, c't Fotografie, Mac & i, Make, heise autos). Each loads its own RSS, merges into the heise.de source, and appears as an extra category in the filter panel.

Categories & filters

Category chips are generated dynamically from the articles currently loaded for the active source. New sources start with all visible categories enabled; only categories you explicitly turn off are remembered, per source. Use Select all / Deselect all for quick bulk changes.

No chips showing? The source fetch probably returned no items — chips only exist for loaded articles. See Troubleshooting → categories.

Sorting articles

The Sortierung panel offers: newest first (default), favorites first / only, hidden/read only, a single calendar day, a date range, most comments, most green (positive) or most red (negative) ratings.

Comment- and rating-based sorts need the dev server to fetch forum metadata, and only some sites expose it (mainly Heise and Golem). Those requests are rate-limited by the source.

Article cards: badges, thumbnails, reordering

  • State badges — New (added since last refresh), Seen (opened), Read (marked read). Their colors are customizable (see Appearance).
  • Thumbnails — feed-provided images, toggleable in settings.
  • Drag & drop — reorder cards within the current view; the manual order persists for that view.

Alternative links, Reddit & YouTube

  • Alternative article links — real third-party URLs for the same story, fetched from Bing News via the dev server (0–15, with an optional domain blacklist and expanded/collapsed display). These are not AI-invented links.
  • Reddit threads — related discussions; when LM Studio is running the server can AI-optimize the search queries. Discovery is on-click or always.
  • YouTube suggestions — the AI proposes searches and the app opens YouTube result pages; no YouTube API key required for that path.
  • Web search engine — pick DuckDuckGo, Google, or Bing for the card's “More articles” button.

Article translation

Enable machine translation of the article area to translate titles, categories, AI summaries, alternative links, and Reddit text via the MyMemory API into your chosen target language. Article links can be wrapped through Google or Microsoft translation. The free MyMemory tier has a daily quota.

Background refresh & bulk generation

  • Update interval — auto-refresh the current source every 5–480 minutes (default 60).
  • Background refresh — optionally prewarm all enabled (or just favorite) sources on that interval, including summaries, alternative links, and Reddit results.
  • Selected-sources generation (“Alle Artikel aus allen Quellen erneuern”) — a manual run with its own modal: choose scope and time period, watch progress/ETA, and cancel anytime.
  • Summarize all — generate AI summaries for every article matching the current filter (respecting cache and concurrency).

Export & AI statistics

Export (header ⤓) writes CSV, JSON, or XLSX for selected articles or a time range, optionally including AI summaries, alternative links, Reddit threads, and AI metadata. Articles can also be exported to PDF with thumbnails.

KI statistics (in KI-Server settings → Statistik…) shows summary count, total and average tokens, average processing time, a usage chart, and your most-used model, with a clear-stats button.

6 Appearance & Themes

Light/dark mode & accent colors

Under Darstellung & Farben pick a theme mode (System, Light, Dark) and one of eight accent colors: heise (orange), ocean, forest, violet, amber, rose, slate, midnight. The accent recolors buttons, links, and highlights throughout the UI. (This manual follows the same accent and mode you set in the app.)

Brightness & transparency sliders

  • Surface brightness (light / dark) — 50 is neutral; below darkens surfaces, above lightens them. A perceptually balanced curve keeps text contrast readable automatically.
  • Header transparency (light / dark) — 0 is opaque, 100 fully transparent, controlling how much of the banner art shows through.

Custom surface & header colors

Color pickers let you override the primary/secondary/card backgrounds and borders for both light and dark modes, and separately the header bar's surface, text, and border. The brightness sliders apply on top of your custom colors.

Article-state badge colors & reset

Set the colors for the New, Seen, and Read badges (defaults: green #2d8a3e, amber #d9a20b, gray #6b7280). A “Restore defaults” button resets all theme and color settings at once.

8 Troubleshooting

Feeds empty or showing demo articles

  • Make sure you opened the app from the dev server (http://127.0.0.1:8080/…), not by double-clicking index.html.
  • Many publisher RSS feeds don't send CORS headers, so they need the dev server proxy. International sources depend on /api/source-feed.
  • Open the console (F12) to see the exact network error.

Categories missing or unexpected

Chips are built from loaded articles — if the fetch returned nothing, no chips appear. New categories start enabled; only your explicit deactivations persist. After a code update, hard-reload the page so the latest scripts load.

AI: “Failed to fetch” or LM Studio logs OPTIONS /api/v1/chat

  • Use python3 scripts/dev_server.py and enable REST via same page origin in KI settings, then reload.
  • Alternatively enable CORS in LM Studio, or use scripts/lm_studio_cors_proxy.py.
  • HTTP 400 usually means a wrong model id or an unsupported reasoning value — the client retries without reasoning when it can.

Content-Security-Policy & remote AI hosts

  • This manual's Ask the AI can reach the same-origin dev-server proxy, local LM Studio (127.0.0.1/localhost), and api.anthropic.com. A custom remote OpenAI-compatible host is blocked by the page's static CSP — use LM Studio same-origin or Anthropic for the manual's AI box.
  • Mixed content: serve the app over http://localhost/127.0.0.1 when talking to a local HTTP model API, or terminate TLS in front of both.

OpenAI / Anthropic base-URL & auth errors

  • 'messages' field is required → your OpenAI base URL includes the path. Use http://host:port/v1, not …/v1/chat/completions.
  • Anthropic base URL must be https://api.anthropic.com/v1 (client appends /messages). A missing/invalid key gives 401/403; reload the model list if Automatic can't resolve a model.

Reset the cache / start fresh

Summaries and settings live in IndexedDB + localStorage. Clearing site data removes them. To hard-reset from the console (destructive):

localStorage.clear();
indexedDB.deleteDatabase('HeiseNewsDashboard');
caches.keys().then((n) => n.forEach((x) => caches.delete(x)));
location.reload();

Comment counts missing? They come from dev-server routes; a plain python -m http.server won't provide them.

9 Privacy & Data Flows

Where your data goes

  • Local storage — articles, settings, and AI summaries stay in your browser (IndexedDB + localStorage).
  • Publisher feeds — fetching news requests public RSS/Atom from third-party sites, which may log requests per their policies.
  • AI providers — a summary request sends the article URL, title, and feed description (and your API key if required) to the endpoint you configured (local LM Studio, an OpenAI-compatible API, or Anthropic). Their privacy policy applies.
  • Dev-server helpers — Bing News, Reddit search, and URL checks run server-side; those queries leave your machine to the respective services.
  • No accounts, no telemetry — the project ships no user database or tracking.

10 Adding a News Source (advanced)

Register and wire a new publisher

This is a developer task. In short:

  1. Register the source in js/news-sources-registry.js with at least id, siteUrl, displayName, language, and a feedStrategy (bing_news, direct_rss, or google_news).
  2. Scraping: registry-driven sources map to /api/source-feed automatically; add a configureSource branch in js/news-scraper.js only for bespoke handling.
  3. Dev server: prefer the shared /api/source-feed; add a dedicated route only if needed, and document it.
  4. CSP: if the browser should fetch the publisher directly, add its host to connect-src in index.html.
  5. UI / i18n: add a brand logo URL in getBrandLogoUrl and header.news_source_* keys in both locale files.
  6. AI metadata: extend the id/host/label maps in js/ai-summarizer.js so prompts and alternative-link filtering use the right domain.

The repository's docs/adding-sources.adoc has the full checklist.

11 FAQ

Frequently asked questions

Do I have to run the Python dev server?

Practically, yes. Without it most feeds fail (CORS), the international catalog and helper routes don't work, and same-origin AI proxying is unavailable.

Do I need an API key?

No — with LM Studio (local) everything is free and offline. A key is only needed for the Anthropic (or a keyed OpenAI-compatible) mode.

Which AI mode should I choose?

Private & free → LM Studio. Best cloud quality → Anthropic Claude. Already running Ollama/vLLM → OpenAI-compatible.

Does it work fully offline?

The app and local AI work offline, but fetching news and helper data (feeds, Bing News, Reddit) needs internet.

Why does “Ask the AI” in this manual say it can't reach my model?

You have no AI backend configured, or it's unreachable. Configure KI-Server in the app first — the manual reuses those exact settings. See AI / KI Setup.

12 Glossary

Terms used in this manual

  • KI — German for “AI” (Künstliche Intelligenz); the app's AI settings dialog is labelled KI-Server.
  • LM Studio — a desktop app that runs open-weight language models locally and exposes a REST server.
  • RSS / Atom — XML feed formats publishers use to syndicate articles.
  • IndexedDB — the browser database where the app stores articles, summaries, and settings.
  • Reasoning — a model's optional internal “thinking” mode; the app shows only the final answer, never the reasoning.
  • CSP — Content-Security-Policy, a browser rule set controlling which hosts a page may contact.
  • venv — a Python virtual environment that isolates installed packages from the system.
  • CORS / preflight — browser cross-origin rules; the dev server proxy sidesteps them.
