Multi-source, responsive news dashboard (Heise, Telepolis, Golem, ComputerBase, t3n, IT-Administrator, The Verge) with optional *AI summaries* (LM Studio REST v1 or OpenAI-compatible APIs), local IndexedDB cache, and a Python dev server for feeds, comments metadata, and same-origin KI proxying.

License: link:LICENSE[MIT] (see <<license,License>>). Feed content and trademarks belong to their respective publishers.

toc::[]

== Quick start (recommended)

[source,bash]
----
cd news_reader_web
python3 scripts/dev_server.py
# Open http://127.0.0.1:8080/index.html
----

The bundled server serves static files and proxies LM Studio (`POST /api/v1/chat`, `GET /v1/models`) plus feed and helper routes (see link:docs/dev-server-api.adoc[Dev server API]).

=== KI (LM Studio)

. Start LM Studio local server (e.g. port `1234`).
. Open *KI-Server* in the dashboard: mode *LM Studio REST v1*, base URL `http://127.0.0.1:1234` (no path), model name exactly as shown in LM Studio.
. Enable *REST via same page origin* when using `dev_server`, save, reload.

Details: link:docs/ki-integration.adoc[AI integration guide].

== Plain static server (limited)

[source,bash]
----
python3 -m http.server 8080
----

Without `dev_server`, many `/api/*` routes and same-origin KI proxying are unavailable; some feeds may fail in the browser due to CORS.
