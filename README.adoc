Multi-source, responsive news dashboard with optional *AI summaries* (LM Studio REST v1 or OpenAI-compatible APIs), local IndexedDB cache, and a Python dev server for feeds, comments metadata, and same-origin KI proxying. Besides the original tech/news sources (Heise, Telepolis, Golem, ComputerBase, t3n, IT-Administrator, BILD, The Verge), the source catalog now also includes a broad international publisher list in `js/news-sources-registry.js` (for example Reuters, AP News, BBC News, CNN, New York Times, Guardian, Washington Post, Wall Street Journal, Bloomberg, Financial Times, Fox News, USA Today, Politico, Deutsche Welle, Der Spiegel, WELT, SГјddeutsche Zeitung, FAZ, Tagesschau, Le Monde, El PaГ­s, Nikkei, CBC News, ABC Australia, Al Jazeera and many more).

== Quick start (recommended)

[source,bash]
----
cd news_reader_web
python3 scripts/dev_server.py
# Open http://127.0.0.1:8080/index.html
----

The bundled server serves static files and proxies LM Studio (`POST /api/v1/chat`, native `GET /api/v1/models`, fallback `GET /v1/models`) plus feed and helper routes (see link:docs/dev-server-api.adoc[Dev server API]). Many of the new international sources are fetched through the shared `/api/source-feed` route, which builds source-specific RSS streams from Bing News search results or proxies Google News RSS.

=== Current UI highlights

* The source selector in the header is sorted alphabetically by the visible source label.
* The header offers previous/next source navigation plus a separate favorite-source jump button; favorite sources can be toggled directly next to the active source logo/wordmark.
* The settings dialog controls which sources appear in the header; favorite sources are stored separately from the enabled-source list.
* Category chips are generated dynamically from the loaded articles of the active source. New sources start with all visible categories enabled; only explicit user deactivations are persisted, per source.
* Optional background refresh can prewarm all enabled sources on the chosen interval. The manual *selected sources generation* run opens its own status modal and processes AI summaries, alternative links, and Reddit discovery with progress, cancel, and ETA feedback.

=== KI (LM Studio)

. Start LM Studio local server (e.g. port `1234`).
. Open *KI-Server* in the dashboard: mode *LM Studio REST v1*, base URL `http://127.0.0.1:1234` (no path), model either explicitly selected from the LM Studio list or left on *No specific choice вЂ” server decides*.
. Enable *REST via same page origin* when using `dev_server`, save, reload.
. The *Reasoning* controls are model-aware in LM Studio mode: the UI only shows levels reported by the current model via `/api/v1/models`, disables the toggle for models without reasoning support, and hides the controls entirely in *OpenAI-compatible* mode.

Details: link:docs/ki-integration.adoc[AI integration guide].

== Plain static server (limited)

[source,bash]
----
python3 -m http.server 8080
----

Without `dev_server`, many `/api/*` routes and same-origin KI proxying are unavailable; some feeds may fail in the browser due to CORS. The new registry-backed international sources depend on `/api/source-feed`, so for the full source catalog `scripts/dev_server.py` is the intended runtime path.
