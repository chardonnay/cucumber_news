== Adding a new article source

Follow these steps to integrate another RSS/Atom publisher in a way consistent with the existing codebase.

image::docs/images/diagram-adding-news-source.png[Flow for adding a news source,width=800,align=center]

=== 1. Register the source

Edit `js/news-sources-registry.js`:

* Add a registry row with at least `id`, `siteUrl`, `displayName`, and `language`.
* Add `hosts` when the article URLs use aliases different from the main `siteUrl` host.
* For registry-driven sources, prefer `feedStrategy`:
** `bing_news` for the shared Bing News → synthetic RSS path
** `direct_rss` for an official RSS URL proxied by the dev server
** `google_news` if you explicitly want the generic Google News RSS path
* Add `searchSites` for `bing_news` if the publisher needs multiple host/path hints.
* Add `feedUrl` for `direct_rss`, plus an optional `feedVersion` when you need a cache-busting revision marker.

Use a stable `id` (snake_case matches existing entries, e.g. `it_administrator`).

The settings checklist follows registry order, but the header dropdown currently sorts *enabled* sources alphabetically by the visible localized label.

=== 2. Configure scraping

For many sources, the registry entry is enough and no extra `configureSource` branch is needed. `NewsScraper` already maps registry rows with `feedStrategy` to `GET /api/source-feed`.

Add a dedicated `configureSource` branch in `js/news-scraper.js` only if the source needs bespoke handling, for example:

* a dedicated proxy route such as `/api/bild-feed`
* custom host matching or multi-feed composition
* non-standard category inference
* special thumbnail extraction or advertorial filtering
* a dedicated comment helper route

If you do add source-specific parsing logic:

* Atom → extend `parseAtomFeed`
* RSS 2.0 → extend `parseRss2Feed`
* emit meaningful `category` and, where possible, `categoryName` values — the visible category chips in the UI are generated dynamically from the loaded article items of the active source

Add a fallback demo item in `getFallbackNews` and ensure `clearAllFeedCaches` knows the cache name when you introduce a new dedicated cache.

=== 3. Dev server proxy (if needed)

In `scripts/dev_server.py`:

* Prefer the shared `GET /api/source-feed` path for new publishers where `bing_news` or `direct_rss` is sufficient.
* Add a dedicated cached fetch helper and `GET /api/your-feed` route only when the shared source-feed proxy is not enough.
* If the source needs comment metadata, add `GET /api/your-comments` (full parser or stub/metadata route, depending on what the site exposes).
* Document new routes in both the server banner string and `docs/dev-server-api.adoc`.

=== 4. Content Security Policy

If the browser should fall back to a *direct* `fetch` to the publisher, add the host to `connect-src` in `index.html`’s CSP meta tag. If you only ever use the same-origin proxy, you may omit the external host.

=== 5. UI, i18n, and migrations

* `js/app.js` — brand logo URL in `getBrandLogoUrl`, optional wordmark/alt branches in locale application, and, if required, a *catalog migration* bump (`newsSourcesCatalogMigrationVersion`) so existing users get the new id in `enabledNewsSources`.
* `locales/de.json` and `locales/en.json` — `header.news_source_your_id`, and optional `brand_wordmark_*` / `logo_alt_*` keys.
* No dedicated filter-group wiring is normally required anymore; category chips are derived from article data at runtime.

=== 6. AI summarizer metadata

Edit `js/ai-summarizer.js`:

* Include the new id in `newsCatalogIds()` fallback array (when `window.NEWS_SOURCE_IDS` is missing).
* Extend `newsSourceIdMatchesHostname` and `getNewsSourceLabel()` maps so alternative-link filtering and prompts use the correct domain.
* If the site is German/English, extend `siteLanguageFromArticleUrl` when it is not already covered by defaults.

=== 7. Manual test checklist

* Load via `python3 scripts/dev_server.py`, select the new source, confirm articles appear.
* Confirm the source shows up in the settings checklist and, when enabled, in the header dropdown.
* Confirm visible categories are generated from the loaded articles and start enabled by default.
* Toggle summaries with LM Studio running (same-origin REST).
* Open an article with “alternative links” and verify `/api/search-news` returns third-party URLs only.
* If the source uses a dedicated feed route, call that route directly and confirm it returns valid RSS/Atom.
