== Troubleshooting

=== News feed empty or fallback demo articles

* Open the browser developer console and check for network errors.
* *Heise* and some other feeds require correct URLs; many non-Heise RSS feeds need `scripts/dev_server.py` because publishers do not send `Access-Control-Allow-Origin` for RSS.
* Many registry-backed international sources depend on `GET /api/source-feed`; if you open the app without `dev_server`, or if the underlying source proxy/search returns no usable items, the UI can fall back to the offline demo entry for that source.
* Confirm you opened the app from the dev server origin when using `/api/*-feed` or `/api/source-feed` proxies.

=== Categories missing or unexpectedly disabled

* Category chips are built from the articles currently loaded for the active source. If the source fetch fails or returns no items, no category chips are shown.
* New sources and new categories start enabled by default. Only explicit user deactivations are persisted, per source.
* If the visible chips do not match the latest code after an update, hard reload the page so the current `index.html` script versions are used.

=== AI: “Failed to fetch” or LM Studio logs `OPTIONS /api/v1/chat`

* Prefer `python3 scripts/dev_server.py` and open `http://127.0.0.1:8080/index.html` (or your chosen port).
* In *KI settings*, enable REST via the same origin and save; reload the page.
* Alternative: `scripts/lm_studio_cors_proxy.py` or LM Studio *Enable CORS* (see LM Studio documentation).

=== Mixed content (HTTPS page, HTTP API)

* Serve the dashboard over `http://localhost` / `http://127.0.0.1` for local LM Studio, or terminate TLS in front of both app and API.

=== OpenAI-compatible mode: `'messages' field is required`

* Base URL must be `http://host:port/v1` — not `…/v1/chat/completions` (the client appends the path).

=== Anthropic mode: authentication or model errors

* Base URL must be `https://api.anthropic.com/v1` (or another Anthropic-compatible `/v1` base URL) — not `…/v1/messages` (the client appends the path).
* Enter an Anthropic API key; `GET /v1/models` and `POST /v1/messages` require it.
* If *Automatic* cannot resolve a model, reload the model list in *KI-Server* or select a model id explicitly.

=== Reasoning options do not match the selected LM Studio model

* The model-aware reasoning UI depends on native `GET /api/v1/models`. If the app can only reach the older OpenAI-style `GET /v1/models`, capability metadata is missing and the UI falls back conservatively to `Off` only.
* Use *LM Studio REST v1* mode for model-aware reasoning controls. In *OpenAI-compatible* and *Anthropic* modes the reasoning block is intentionally hidden.
* Reload the model list in the KI settings dialog after changing the LM Studio server, model, or same-origin setting.
* If the browser still shows stale options after a code update, hard reload the page so the current script versions are used.

=== Summary cache

* Summaries live in IndexedDB; clearing *site data* removes them.
* To hard-reset in the console (destructive):

[source,javascript]
----
localStorage.clear();
indexedDB.deleteDatabase('HeiseNewsDashboard');
caches.keys().then((names) => names.forEach((n) => caches.delete(n)));
location.reload();
----

Adjust the database name if your fork renamed it in `storage.js`.

=== Comment counts missing

* Comment integration depends on `dev_server` routes. Pure `python -m http.server` will not provide `/api/heise-comments` or `/api/golem-comments`.
